**SCREEN 1 — Introduction & Setup**  
**Goal:**  
Help the user understand the purpose of the experience, choose a challenge, and define what kinds of insights they want to explore.

**What the user sees:**

1. **Start button** — simple entry point: *“Start presentation.”*  
2. After clicking, two **introductory sentences** appear in 2 rows, explaining the context: “Collective intellegance, on your terms.” “Different minds, one challenge.”  
3. **Challenge selection** — 2 cards (e.g., *Emergency Response*, *Public Transit*).  
4. **Insight-type selection** — four selectable chips:  
   **Perspectives • Tensions • Opportunities • Blind Spots.**  
   *(User can choose one or several.)*  
5. Final message in 2 rows: “Let's examine this from different angles.”, "We'll gather specialists with diverse perspectives."  
6. **Button:** *“Assemble the team.”* → transitions to screen 2\.

**What the user does:**

1. Click *Start presentation*.  
2. Read short intro text.  
3. Select a **challenge**.  
4. Choose desired **insight types**.  
5. Click *Assemble the team.*

**Output:**  
selectedChallenge \+ selectedInsights.  
(Then we transfer it to screens 2–4 to highlight the corresponding content.)

**SCREEN 2 — Team Assembly**  
**Goal:**  
Show how the system builds a diverse team of specialists and personalities, illustrating the logic behind team composition.

**What the user sees:**

1. A single agent card appears in the center.  
2. Step 1: specialization shown → left **footnote** explains the role.  
3. Step 2: personality revealed → right **footnote** explains why diverse mindsets are needed.  
   *(Footnotes remain visible until the user continues.)*  
4. Button: *“Continue.”*  
5. After clicking: the full team appears around the center, then the button *“Start Discussion.”* What team will be shown depend on the selectedChallenge.

**What the user does:**

1. Observe how one agent is formed.  
2. Read both footnotes.  
3. Click *Continue* → watch the full team appear.  
4. Click *Start Discussion* → go to screen 3\.

**Output:** none

**SCREEN 3 — Discussion Network**  
**Goal:**  
Visualize how the agents interact, debate, and exchange ideas to surface different viewpoints.

**What the user sees:**

1. A dynamic graph of agents (nodes colored by personality).  
2. 3–4 chat icons floating above active connections.  
3. Instruction line at the top: “Agents are discussing. Tap a conversation to peek inside.”  
4. When a chat is opened: right-side thread viewer with short dialogue.  
5. After at least one thread is viewed: *“Show results”* button appears.

**What the user does:**

1. Open one or more conversation threads.  
2. Read short exchanges.  
3. Click *Show results* to move on 4th screen.

**Output:** none.

**SCREEN 4 — Insights (Results)**  
**Goal:**  
Present the key perspectives and takeaways that emerged from the agents’ discussion, based on the user’s selected insight types.

**What the user sees:**

1. Calm layout with a title: *“What the team surfaced.”*  
2. 3–4 insight cards corresponding to the user’s earlier selections:  
   **Perspectives, Tensions, Opportunities, Blind Spots.**  
3. Each card: short title \+ 1-line explanation \+ small icons representing the card.  
4. At the bottom: message inviting reflection, e.g. “Which of these viewpoints challenges your own assumptions?”  
5. Button: *“Explore how this could work for your organization.”* → next 5th screen.

**What the user does:**

1. Read through the insight cards.  
2. Click *Explore how this could work for your organization.*

**Output:** none.

**SCREEN 5 — Real-World Scenarios**  
**Goal:**  
Demonstrate how the same framework applies across real organizational contexts through clear, relatable case examples.

**What the user sees:**

1. Title: *“See how this framework applies across different domains.”*  
2. Grid of case cards (2×3 or 3×3):  
   *Simulating phsycology of some group of people, forum for discussing problems from different angles, dialoque* etc.  
3. When a card is opened: deyailed explanation of the case  
4. User can browse freely between cases.

**What the user does:**

1. Select one or several case cards.  
2. Open and read detailed examples.

**Output:** none.
